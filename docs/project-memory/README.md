# Протокол памяти проекта

Обновлено: 2026-05-17

## Назначение

Этот протокол защищает контекст проекта, если чат ChatGPT или Codex стал слишком длинным, сломался во время compact, пропал из поиска или работу нужно перенести в новый чат.

Главное правило:

**Чат не является источником правды. Источник правды — папка проекта.**

## Какие программы использовать

- `Codex Desktop`: основная программа для работы с проектом, файлами и ассистентом.
- `ChatGPT`: используйте только если конкретный проект или чат ведется в ChatGPT.
- `Terminal`: создание папок, запуск проверок, просмотр статуса проекта.
- `Finder`: ручной поиск папок проекта на macOS.
- `GitHub Desktop` или `git` в Terminal: резервное сохранение изменений, если проект ведется в git.

## Стандартный “черный ящик” проекта

```text
docs/project-memory/
  README.md
  CONTEXT_HEALTH.md
  PROJECT_STATE.yaml
  HANDOFF.md
  WORKLOG.md
  NEXT_ACTIONS.md
  RISKS.md
  ARTIFACTS.md
  DECISIONS/
  PROMPTS/
  RUNS/
```

## Что хранит каждый файл

- `CONTEXT_HEALTH.md`: риск потери контекста и готовность к переходу в новый чат.
- `PROJECT_STATE.yaml`: короткий структурированный статус проекта.
- `HANDOFF.md`: готовая передача контекста в новый чат.
- `WORKLOG.md`: фактический журнал выполненных работ.
- `NEXT_ACTIONS.md`: ближайшие конкретные действия.
- `RISKS.md`: блокеры, неизвестные, слабые предположения.
- `ARTIFACTS.md`: важные файлы, ссылки, outputs, скриншоты, отчеты.
- `DECISIONS/`: архитектурные, продуктовые и процессные решения.
- `PROMPTS/`: переиспользуемые промпты для агентов и инструментов.
- `RUNS/`: статусы парсеров, фоновых задач и долгих процессов.

Не храните здесь секреты, API-ключи, пароли, приватные токены и клиентские доступы.

## Как начать новый чат

В новом чате отправьте:

```text
Продолжи этот проект из файлов, а не из памяти прошлого чата.

Сначала прочитай:
1. AGENTS.md
2. docs/project-memory/CONTEXT_HEALTH.md
3. docs/project-memory/PROJECT_STATE.yaml
4. docs/project-memory/HANDOFF.md
5. docs/project-memory/NEXT_ACTIONS.md
6. docs/project-memory/WORKLOG.md

Не выдумывай содержимое старого чата.
Если факт не подтвержден файлами проекта или моим сообщением, помечай его как гипотезу.

Первым ответом дай:
1. текущий статус проекта
2. уровень риска контекста
3. подтвержденное следующее действие
4. риски или недостающий контекст
```

## Когда обновлять project memory

Обновляйте project memory:

- после каждой завершенной функции;
- после каждого исследовательского отчета;
- после каждого parser batch или checkpoint долгой задачи;
- перед переходом в новый чат;
- перед остановкой работы на день;
- после важных решений пользователя;
- после ошибки, которая меняет следующий шаг.
